1 Présentation du projet
1.1 Contexte
Dans le cadre du partenariat établi entre Yazaki Morocco Meknès (YMM) et
l’École Nationale Supérieure des Arts et Métiers de Meknès (ENSAM Mek-
nès), une compétition industrielle est organisée afin de sélectionner quatre étudiants pour
un stage au sein de l’entreprise. Le projet consiste à concevoir et développer une plate-
forme web centralisée dédiée au suivi et à la gestion des DFC (Documents de Faisabilité
de Changement) et des ECO (Engineering Change Orders), dans une démarche de digi-
talisation adoptée par l’entreprise, visant à optimiser les processus, améliorer l’efficacité
opérationnelle et renforcer la traçabilité.
1.2 Problématique
Actuellement, la gestion des DFC chez Yazaki Morocco Meknès nécessite une solution
digitale permettant de :
— Centraliser les DFCs
— Suivre le statut de faisabilité de chaque DFC
— Gérer les documents associés (rapports, dérogations)
— Visualiser les statistiques et indicateurs de performance
— Assurer la traçabilité complète du processus
1.3 Objectifs du projet
O1. Développer une application web responsive permettant la gestion complète des DFC
O2. Mettre en place un système de formulaires pour la saisie et le suivi des DFC
O3. Créer des dashboards analytiques pour les utilisateurs et administrateurs
O4. Implémenter un système de gestion des utilisateurs
O5. Assurer la traçabilité
2 Périmètre fonctionnel

L’application comprendra deux interfaces principales :
— Interface Utilisateur : pour la saisie et le suivi des DFC
— Interface Administrateur : pour la supervision globale et la gestion des utilisa-

Méthode 2 : Import et extraction automatique
L’utilisateur peut uploader un fichier DFC/Dérogation existant (Excel, pdf) et le système
extrait automatiquement les données pour pré-remplir le formulaire. Cette fonctionnalité
utilise :
— OCR (Reconnaissance Optique de Caractères) pour les PDF scannés
— Extraction de texte pour les PDF natifs
— Parsing Excel/CSV pour les fichiers tableurs
— Intelligence Artificielle pour identifier et mapper les champs automatiquement
Le processus d’extraction automatique :
1. L’utilisateur upload le fichier DFC/Derogation
2. Le système détecte le format et extrait le texte
3. L’IA identifie les champs pertinents (N° DFC, Projet, Description, N°Dérogation,
etc.)
4. Le formulaire est pré-rempli avec les données extraites
5. L’utilisateur vérifie et corrige si nécessaire avant validation

3 Critères d’acceptation
3.1 Critères fonctionnels
Le projet sera considéré comme réussi si :
1. Tous les formulaires fonctionnent correctement avec validation
2. Les DFC peuvent être créés, modifiés, consultés et supprimés
3. Les dashboards affichent les bonnes statistiques en temps réel
4. Les filtres du dashboard administrateur fonctionnent
5. Le système d’authentification
3.2 Critères de qualité
1. Interface intuitive et ergonomique
2. Navigation fluide et cohérente
3. Messages d’erreur clairs
4. Documentation complète et claire
5. Code propre et structuré (bonnes pratiques)
